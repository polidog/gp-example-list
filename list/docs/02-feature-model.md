# フィーチャーモデル: リストコンテナ

## フィーチャーツリー

```
ListContainer
├── ElementType [変化領域・パラメータ型]
│   （要素の型を指定するパラメータ）
├── Storage [変化領域・XOR]
│   ├── ArrayBased [有効・差し替え可能]
│   │   ├── InitialCapacity [有効・パラメータ] = 16
│   │   └── GrowthStrategy [変化領域・XOR]
│   │       ├── Doubling [有効・差し替え可能]
│   │       └── Additive [無効・差し替え可能]
│   └── LinkedList [無効・差し替え可能]
│       └── Direction [変化領域・XOR]
│           ├── Singly [有効・差し替え可能]
│           └── Doubly [無効・差し替え可能]
├── CoreOperations [固定]
│   ├── Add [有効・固定]
│   ├── Get [有効・固定]
│   └── Size [有効・固定]
└── OptionalOperations [変化領域・OR]
    ├── Remove [有効]
    ├── Insert [有効]
    ├── Search [変化領域・OR]
    │   ├── LinearSearch [有効]
    │   └── Contains [有効]
    ├── Sort [無効]
    │   └── SortAlgorithm [変化領域・XOR]
    │       ├── InsertionSort [無効・差し替え可能]
    │       └── MergeSort [無効・差し替え可能]
    └── Iteration [有効]
        └── Direction [変化領域・XOR]
            ├── Forward [有効・差し替え可能]
            └── Reverse [無効・差し替え可能]
```

凡例:
  [有効・固定]         — 現在有効で、変わる見込みがない
  [有効・差し替え可能]  — 現在有効だが、将来差し替える可能性がある
  [有効]               — 現在有効
  [無効]               — 現在無効だが、有効にできる
  [変化領域・XOR]      — 子フィーチャーから1つだけ選択
  [変化領域・OR]       — 子フィーチャーから1つ以上選択

## フィーチャー詳細

### ElementType
- **現在の状態**: 有効
- **変化の可能性**: パラメータ型
- **変化の駆動要因**: 利用シーンごとに格納する型が異なる
- **説明**: リストに格納する要素の型。言語がジェネリクスをサポートする場合は型パラメータとして、そうでない場合は具体型として指定する
- **パラメータ**: 型名（例: `int`, `string`, `any`）

### Storage（データ構造）

#### ArrayBased
- **現在の状態**: 有効
- **変化の可能性**: 差し替え可能
- **変化の駆動要因**: 性能要件（ランダムアクセスが重要か、挿入・削除が重要か）
- **説明**: 連続したメモリ領域に要素を格納する。インデックスによるO(1)アクセスが可能
- **パラメータ**: なし

#### InitialCapacity
- **現在の状態**: 有効
- **変化の可能性**: パラメータ型
- **変化の駆動要因**: 予想される要素数に応じて最適化
- **説明**: 配列ベース実装における初期確保容量
- **パラメータ**: 正の整数（デフォルト: 16）

#### GrowthStrategy
- **現在の状態**: 有効（Doubling）
- **変化の可能性**: 差し替え可能
- **変化の駆動要因**: メモリ効率 vs 再割当て頻度のトレードオフ
- **説明**: 配列の容量が不足した場合の拡張戦略

#### LinkedList
- **現在の状態**: 無効
- **変化の可能性**: 差し替え可能
- **変化の駆動要因**: 頻繁な挿入・削除が必要な場合
- **説明**: 各要素がポインタで次（/前）の要素を参照する実装

#### Direction（LinkedList）
- **現在の状態**: 無効（Singly がデフォルト）
- **変化の可能性**: 差し替え可能
- **変化の駆動要因**: 逆方向走査や末尾操作の効率が必要な場合
- **説明**: 単方向（Singly）か双方向（Doubly）かの選択

### CoreOperations（基本操作）

#### Add
- **現在の状態**: 有効・固定
- **変化の可能性**: 固定
- **説明**: リストの末尾に要素を追加する

#### Get
- **現在の状態**: 有効・固定
- **変化の可能性**: 固定
- **説明**: インデックスを指定して要素を取得する

#### Size
- **現在の状態**: 有効・固定
- **変化の可能性**: 固定
- **説明**: リストの現在の要素数を返す

### OptionalOperations（オプション操作）

#### Remove
- **現在の状態**: 有効
- **変化の可能性**: 追加型
- **変化の駆動要因**: 追加専用リストには不要
- **説明**: インデックスを指定して要素を削除する

#### Insert
- **現在の状態**: 有効
- **変化の可能性**: 追加型
- **変化の駆動要因**: 末尾追加のみで十分な場合は不要
- **説明**: 指定位置に要素を挿入する

#### Search
- **現在の状態**: 有効
- **変化の可能性**: 追加型
- **変化の駆動要因**: 検索が不要な用途もある
- **説明**: リスト内の要素を検索する機能群

#### Sort
- **現在の状態**: 無効
- **変化の可能性**: 追加型
- **変化の駆動要因**: 順序が重要でない場合は不要
- **説明**: リストの要素をソートする。アルゴリズムを選択可能

#### Iteration
- **現在の状態**: 有効（Forward）
- **変化の可能性**: 追加型
- **変化の駆動要因**: 走査方向の要件
- **説明**: リストの要素を順番に走査する。方向を選択可能

## 現在のシステム構成

```
ListContainer
  ElementType = int
  Storage = ArrayBased
    InitialCapacity = 16
    GrowthStrategy = Doubling
  CoreOperations = [Add, Get, Size]
  OptionalOperations = [Remove, Insert, Search(LinearSearch, Contains), Iteration(Forward)]
```
