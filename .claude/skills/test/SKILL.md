# テスト生成

生成コードの正しさを検証するテストを生成・実行する。

## 引数

- テスト対象の生成コードのディレクトリ、またはDSL仕様書のパス。

## 手順

1. `docs/gp/09-configuration-knowledge.md` のテスト性質セクションを読む
2. 生成コードのAPIを確認する
3. 共通性質（全構成で成立）と構成固有の性質をテストコードとして生成する
4. テストを実行し、失敗があれば生成コードまたはテストを修正する
