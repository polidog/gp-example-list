# ドメインスコーピング

システムが何を解決し、誰のためのもので、どこまでが対象範囲かを明確にするスキル。
ジェネレーティブプログラミングのドメイン分析活動。

## 手順

### ステップ1: システムの目的の明確化

- ユーザーに対象システムの概要を確認する
- システムが**解決する問題**を特定する
- **主要な利用者**（ステークホルダー）とその目的を把握する
- **ビジネスの文脈**（市場、競合、制約条件）を確認する

### ステップ2: ドメインの範囲の確定

- システムが扱う**対象範囲（スコープ）**を定義する
- **スコープ内**: このシステムが責任を持つ領域
- **スコープ外**: このシステムでは扱わない領域（隣接ドメイン、将来的に検討するが今は対象外など）
- プロジェクト内に既存のコード、ドキュメント、設定ファイルがあれば分析してスコープの参考にする

### ステップ3: サブドメインの識別

- ドメイン内にサブドメイン（関心領域の塊）があれば識別する
- 各サブドメインの役割と、サブドメイン間の関係を整理する
- コアドメイン（競争優位の源泉）と支援ドメイン（必要だが差別化要素ではない部分）を区別する

### ステップ4: ドメイン用語の整理

- ドメイン内で使われる用語を収集する
- 同じ概念に異なる名前が使われていないか確認する
- 用語集を作成する

### ステップ5: ドメインスコープの出力

`docs/gp/01-domain-scope.md` にドメインスコープを出力する。テンプレートとして `templates/domain-scope-template.md` を使用する。

### ステップ6: 次のステップの案内

次の活動として「フィーチャーモデリングと概念モデリング」（`/feature-modeling`）が推奨されるが、必要に応じて他の活動に進んだり、以前の成果物を更新してもよい。
