# 実装

DSL仕様書と構成の知識に基づき、コードを生成する。

## 引数

- DSL仕様書のパス（YAML）。省略時は対話的にフィーチャーを選択する。

## 手順

1. DSL仕様書を読み込み、`go run ./cmd/dsl-validate/ <仕様書> --resolve` でバリデーション・構成解決する
2. `docs/gp/09-configuration-knowledge.md` の生成ルール R1〜R6 に従い、指定言語のコードを生成する
3. `generated/<言語名>/` に出力する
4. コンパイラ/インタプリタが利用可能ならコンパイル・構文チェックを行う

DSLバリデータ自体の実装・修正が必要な場合は、`docs/gp/07-dsl-notation.md` の制約ルールに従い `pkg/dsl/` のコードを更新する。
