# 実装コンポーネントの実装

アーキテクチャ設計と構成の知識に基づき、システムの実装コンポーネントをコードとして実装するスキル。
ジェネレーティブプログラミングのドメイン実装活動。

## 前提確認

- `docs/gp/06-components.md` が存在するか確認する
- `docs/gp/09-configuration-knowledge.md` が存在するか確認する
- 存在しない場合、先に関連する活動を行うことを推奨するが、必要に応じてこの活動から始めてもよい

## 手順

### ステップ1: 実装順序の決定

コンポーネント依存図を参照し、依存関係に基づいて実装順序を決定する。

- 依存される側（下位レイヤー）から順に実装する
- 共通コンポーネント（固定部分）を先に実装する
- 可変コンポーネントはインターフェースを先に定義し、デフォルト実装から着手する

### ステップ2: 共通コンポーネントの実装

フィーチャーモデルで「共通（固定）」とされた部分を実装する。

- コンポーネント定義に記載された責務に従う
- 可変コンポーネントとの接続点（インターフェース、抽象クラス、DI設定等）を確実に設ける
- テストを書く

### ステップ3: 可変コンポーネントの実装

フィーチャーモデルで「可変」とされた部分を実装する。

- まず**インターフェース/契約**を定義する
- 構成の知識で定められた**デフォルト構成の実装**を作成する
- 差し替え可能な設計になっていることを確認する
- テストを書く（インターフェースに対するテストと、具体実装に対するテスト）

### ステップ4: 統合と動作確認

- コンポーネント間の接続を確認する
- デフォルト構成で一通り動作することを確認する

### ステップ5: 次のステップの案内

次の活動としてDSL実装（`/dsl-implementation`）やジェネレータ実装（`/generator-implementation`）との統合が推奨されるが、必要に応じて他の活動に進んだり、以前の活動に戻って成果物を更新してもよい。
