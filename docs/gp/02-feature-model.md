# フィーチャーモデル: リストコンテナ

## フィーチャーツリー

```
LIST [固定]
├── ElementType [必須・変化領域]
│   └── <T> (型パラメータ) [有効・パラメータ型]
├── Ownership [必須・変化領域] ── XOR（1つを選択）
│   ├── ExternalReference [有効・差し替え可能]
│   ├── OwnedReference [有効・差し替え可能]
│   └── Copy [有効・差し替え可能]
├── Morphology [必須・変化領域] ── XOR（1つを選択）
│   ├── Monomorphic [有効・差し替え可能]
│   └── Polymorphic [有効・差し替え可能]
├── LengthCounter [選択・変化領域]
│   └── CounterType [必須（LengthCounter有効時）・変化領域] ── XOR（1つを選択）
│       ├── short [有効・差し替え可能]
│       ├── int [有効・差し替え可能]
│       └── long [有効・差し替え可能]
└── Tracing [選択・変化領域]
    └── (有効 or 無効) [有効・差し替え可能]
```

凡例:
  [固定]                 — 変わる見込みがない
  [有効・差し替え可能]    — 現在有効だが、構成によって差し替える
  [有効・パラメータ型]    — パラメータとして指定する

選択ルール:
  XOR — グループ内から正確に1つを選択
  必須 — 必ず指定が必要
  選択 — 有効/無効を選べる（デフォルト: 無効）

## フィーチャー詳細

### LIST（ルート）
- **現在の状態**: 有効
- **変化の可能性**: 固定
- **説明**: 順序付き要素の集まりを表すコンテナデータ構造。すべてのリストコンテナに共通する基本操作（挿入・削除・走査・検索）を提供する

### ElementType
- **現在の状態**: 有効
- **変化の可能性**: パラメータ型（利用時に型を指定）
- **変化の駆動要因**: 利用者がリストに格納する要素の型は用途ごとに異なる
- **説明**: リストが保持する要素の型。ジェネリック（型パラメータ）として実現し、任意の型を受け取れるようにする
- **パラメータ**: `T` — 要素の型（言語のジェネリクス/テンプレート機構で実現）

### Ownership
- **現在の状態**: 有効
- **変化の可能性**: 差し替え可能（XOR選択）
- **変化の駆動要因**: メモリ管理方針やパフォーマンス要件により最適な所有権モデルが異なる
- **説明**: リストが要素をどのように所有・管理するかを決定するフィーチャー

#### ExternalReference（外部参照）
- **現在の状態**: 有効
- **変化の可能性**: 差し替え可能
- **説明**: リストは要素への参照（ポインタ）を保持するが、要素の生存期間は管理しない。要素のライフサイクルは外部が責任を持つ

#### OwnedReference（所有参照）
- **現在の状態**: 有効
- **変化の可能性**: 差し替え可能
- **説明**: リストは要素への参照（ポインタ）を保持し、要素の生存期間も管理する。リストの破棄時に要素も解放する

#### Copy（コピー）
- **現在の状態**: 有効
- **変化の可能性**: 差し替え可能
- **説明**: リストは要素のコピーを保持する。要素は値として格納され、リスト内に直接埋め込まれる

### Morphology
- **現在の状態**: 有効
- **変化の可能性**: 差し替え可能（XOR選択）
- **変化の駆動要因**: 型の均一性要件により、単一型制約か多態型許容かが変わる
- **説明**: リスト内の要素の型の扱い方を決定するフィーチャー

#### Monomorphic（単相）
- **現在の状態**: 有効
- **変化の可能性**: 差し替え可能
- **説明**: リスト内のすべての要素が同一の型 `T` でなければならない。型安全性が高く、メモリ効率も良い

#### Polymorphic（多相）
- **現在の状態**: 有効
- **変化の可能性**: 差し替え可能
- **説明**: リスト内の要素が `T` のサブタイプであることを許容する。継承階層を活用した柔軟なコレクションが可能

### LengthCounter
- **現在の状態**: 有効（デフォルトは無効）
- **変化の可能性**: 差し替え可能（有効/無効の切り替え）
- **変化の駆動要因**: O(1)での要素数取得が必要かどうかは用途に依存
- **説明**: リストの現在の要素数を常時追跡するカウンタ。有効にすると挿入・削除のたびにカウンタを更新し、要素数の取得がO(1)になる。無効の場合、要素数取得にはO(n)の走査が必要

#### CounterType（カウンタ型）
- **現在の状態**: 有効（LengthCounter 有効時に必須）
- **変化の可能性**: 差し替え可能（XOR選択）
- **変化の駆動要因**: リストの最大要素数に応じて、メモリ効率と表現範囲のトレードオフが生じる
- **説明**: カウンタ値を保持する整数型を指定するフィーチャー。リストの想定最大サイズに応じて適切な型を選択する

##### short
- **説明**: 小さな整数型。メモリ効率が良いが、表現可能な最大要素数が限定的（典型的に最大 32,767 または 65,535）。小規模なリスト向け

##### int
- **説明**: 標準的な整数型。多くの用途で十分な範囲を持つ（典型的に最大約 21億）。デフォルト推奨

##### long
- **説明**: 大きな整数型。非常に大規模なリストに対応できる。メモリ使用量は増加するが、要素数の上限が事実上なくなる

### Tracing
- **現在の状態**: 有効（デフォルトは無効）
- **変化の可能性**: 差し替え可能（有効/無効の切り替え）
- **変化の駆動要因**: デバッグや監視の要件により必要性が変わる
- **説明**: リストへの操作（挿入・削除・検索等）をログとして記録する機能。デバッグや動作確認に有用だが、オーバーヘッドが発生する

## 現在のシステム構成（デフォルト例）

```
LIST
  ElementType = <int>
  Ownership   = Copy
  Morphology  = Monomorphic
  LengthCounter = 無効
  (LengthCounter有効時: CounterType = int)
  Tracing       = 無効
```
