# 概念モデル: リストコンテナ

## エンティティと関係

### List（リスト）
- 説明: 順序付き要素の集まりを管理するコンテナ
- 主な属性: head（先頭要素への参照）、length（要素数、LengthCounter有効時のみ）
- 関連: Element を 0..* 個保持する
- **共通/可変**: 共通（リストというコンテナの存在自体は不変）

### Element（要素）
- 説明: リストに格納される個々のデータ
- 主な属性: value（値、型は ElementType で決定）
- 関連: List に所属する、Node に格納される
- **共通/可変**: 可変
- 変化のタイプ: パラメータ型（ElementType により型が決まる）、駆動要因: 利用者が格納するデータの型は用途ごとに異なる

### Node（ノード）
- 説明: 要素を格納し、リストの連結構造を形成する内部構造体
- 主な属性: data（要素データまたは要素への参照）、next（次ノードへの参照）
- 関連: Element を保持する、次の Node を参照する
- **共通/可変**: 可変
- 変化のタイプ: 構造型、駆動要因: Ownership により data の保持方法が変わり（値埋め込み / ポインタ）、Morphology により型制約が変わる

### LengthCounter（長さカウンタ）
- 説明: リストの現在の要素数を追跡するカウンタ
- 主な属性: count（現在の要素数、型は CounterType で決定: short / int / long）
- 関連: List に付随する（オプショナル）
- **共通/可変**: 可変
- 変化のタイプ: 差し替え型（存在する/しない）+ パラメータ型（カウンタの整数型を選択）、駆動要因: O(1)での要素数取得の要否、およびリストの想定最大サイズによるメモリ効率と表現範囲のトレードオフ

### TraceLog（トレースログ）
- 説明: リスト操作の履歴を記録するログ
- 主な属性: operation（操作種別）、timestamp（タイムスタンプ）、details（詳細情報）
- 関連: List に付随する（オプショナル）
- **共通/可変**: 可変
- 変化のタイプ: 差し替え型（存在する/しない）、駆動要因: デバッグ・監視要件

### エンティティ関連図

```
+--------+       0..*  +--------+       1  +---------+
|  List  |────────────>|  Node  |────────>| Element |
+--------+             +--------+         +---------+
| head   |             | data   |         | value:T |
| length?|             | next   |         +---------+
+--------+             +--------+
    |                      |
    | 0..1                 | Ownership により
    v                      | 保持方法が変化
+---------------+          |
| LengthCounter |    ┌─────┴──────┐
+---------------+    | Copy:      値埋め込み
| count         |    | OwnedRef:  所有ポインタ
+---------------+    | ExtRef:    外部ポインタ
    |                └────────────┘
    | 0..1
    v
+----------+
| TraceLog |
+----------+
| entries  |
+----------+
```

## ユースケース

| ユースケース | 利用者 | 関連エンティティ | 概要 | 共通/可変 | 可変の場合: タイプと駆動要因 |
|-------------|--------|-----------------|------|----------|--------------------------|
| 要素の挿入 | ライブラリ開発者 | List, Node, Element | リストの先頭・末尾・任意位置に要素を追加する | 共通 | — |
| 要素の削除 | ライブラリ開発者 | List, Node, Element | リストから指定位置または条件に合う要素を削除する | 共通 | — |
| 要素の走査 | ライブラリ開発者 | List, Node, Element | リストの先頭から末尾まで順に要素を訪問する | 共通 | — |
| 要素の検索 | ライブラリ開発者 | List, Node, Element | 条件に合う要素をリスト内から見つける | 共通 | — |
| 要素数の取得 | ライブラリ開発者 | List, LengthCounter | リストの現在の要素数を返す | 可変 | 差し替え型: LengthCounter有効時はO(1)、無効時はO(n)走査 |
| 操作のトレース | ライブラリ開発者 | List, TraceLog | リスト操作をログに記録する | 可変 | 差し替え型: Tracing有効時のみ存在 |
| リストの破棄 | ライブラリ開発者 | List, Node, Element | リストとその内部リソースを解放する | 可変 | 差し替え型: Ownershipにより解放責任が異なる |

## ビジネスルール

| ルール | 説明 | 関連要素 | 共通/可変 | 可変の場合: タイプと駆動要因 |
|--------|------|---------|----------|--------------------------|
| 型安全性 | Monomorphic時、リスト内の全要素は宣言された型Tと一致しなければならない | Element, Morphology | 可変 | 差し替え型: Morphologyの選択により制約レベルが変わる |
| サブタイプ許容 | Polymorphic時、Tのサブタイプも格納可能 | Element, Morphology | 可変 | 差し替え型: Morphologyの選択に依存 |
| 所有権の一貫性 | リスト内で所有権モデルは統一されなければならない（要素ごとに混在不可） | Node, Ownership | 共通 | — |
| カウンタ整合性 | LengthCounter有効時、カウンタ値は常に実際の要素数と一致しなければならない | LengthCounter, List | 可変 | 差し替え型: LengthCounter有効時のみ適用 |

## 外部との境界

| 境界 | 種類 | 説明 | 共通/可変 | 可変の場合: タイプと駆動要因 |
|------|------|------|----------|--------------------------|
| 利用者コード | API境界 | リストの公開インターフェース（挿入・削除・走査等のメソッド） | 共通 | — |
| メモリアロケータ | 外部サービス | ノードの動的メモリ確保・解放を担う | 共通 | — |
| トレース出力先 | 外部サービス | Tracing有効時のログ出力先（標準出力、ファイル等） | 可変 | 追加型: 出力先の種類は将来拡張可能 |
| 型システム | 言語機構 | ジェネリクス/テンプレートによる型パラメータの解決 | 可変 | 差し替え型: 生成対象言語により機構が異なる |
